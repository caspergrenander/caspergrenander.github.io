Transparency Notice: This Privacy Policy explains how C / A Beats Publisher handles data. As an internal automation tool, we prioritize data minimization and security in all operations.

1. Introduction

C / A Beats & Productions ("we", "us", "our") operates the C / A Beats Publisher system ("the System"), an internal automation tool for scheduled content publishing to Meta platforms.

This Privacy Policy describes our practices regarding data collection, processing, and protection for the System. We are committed to maintaining the privacy and security of all data processed through our platform.

Core Privacy Principle

Data Minimization: We collect and process only the data strictly necessary for the System's publishing functionality. No external user personal data is collected or stored.

2. Data We Process

2.1 Technical Data

Authentication Data

Purpose: System access & API authorization
Includes: Meta OAuth access tokens
Retention: Until token expiration or revocation

System Logs

Purpose: Security monitoring & debugging
Includes: Timestamps, user actions, API responses
Retention: 30 days for operational purposes

2.2 Content Data

  • Scheduled content (temporary storage during processing)
  • Publishing metadata (timestamps, platform IDs, status)
  • Configuration settings (user preferences for automation)

No External User Data: The System does not collect, store, or process personal data from external users. We only handle data related to our internal publishing operations.

3. How We Use Data

3.1 Primary Purposes

  1. Content Publishing: Processing and scheduling posts to Meta platforms
  2. System Authentication: Verifying authorized user access
  3. API Communication: Interacting with Meta's Graph API
  4. Operation Monitoring: Ensuring system reliability and performance

3.2 Legal Basis for Processing

Data processing is based on:

  • Legitimate Business Interests: Internal content management operations
  • Contractual Necessity: Fulfilling internal service agreements
  • Legal Compliance: Adhering to data protection regulations
  • User Consent: Through Meta's OAuth authorization process

4. Data Security

4.1 Security Measures

  • Encryption: All data transmissions use TLS 1.2+ encryption
  • Access Controls: Role-based permissions and authentication
  • Token Security: Access tokens encrypted at rest
  • Regular Audits: Security assessments and vulnerability scanning
  • Incident Response: Documented procedures for security incidents

4.2 Third-Party Services

The System integrates with:

  • Meta Platform: For content publishing via Graph API
  • Hosting Provider: For system infrastructure

All third-party services are selected based on their security and privacy compliance. Data sharing is limited to what's necessary for System functionality.

5. Data Retention & Deletion

5.1 Retention Periods

Access Tokens

Retained until expiration or revocation
Automatically removed when invalid

System Logs

Retained for 30 days
Anonymized for long-term analytics

Content Data

Processed in real-time
Not stored after publishing

5.2 Data Deletion Requests

Authorized users can request data deletion through:

  • Direct contact with system administrators
  • The Data Deletion Request form
  • Meta's data deletion callback system

Deletion requests are processed within 7-10 business days.

  • Access: Request information about data we process
  • Correction: Request correction of inaccurate data
  • Deletion: Request deletion of your data (with exceptions)
  • Restriction: Request temporary processing restrictions
  • Portability: Request data in a structured format
  • Objection: Object to certain processing activities
